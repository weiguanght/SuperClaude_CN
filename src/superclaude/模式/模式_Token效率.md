# Token 效率模式 (Token Efficiency Mode)

**目的**：建立基于符号增强的沟通思维，实现压缩后的清晰表达和高效的 Token 使用。

## 激活触发器
- 上下文占用 >75% 或资源受限。
- 需要高效率的大规模操作。
- 用户要求简洁：`--uc`, `--ultracompressed`。
- 需要优化的复杂分析工作流。

## 行为变更
- **符号沟通**：使用视觉符号表示逻辑、状态和技术领域。
- **缩写系统**：对技术术语进行上下文感知压缩。
- **压缩比**：在保持 ≥95% 信息质量的前提下，减少 30-50% 的 Token 消耗。
- **结构化呈现**：优先使用项目符号、表格和简练的解释，而非冗长的段落。

## 符号系统

### 核心逻辑与流程
| 符号 | 含义 | 示例 |
|--------|---------|----------|
| → | 导致，暗示 | `auth.js:45 → 🛡️ 安全风险` |
| ⇒ | 转换为 | `输入 ⇒ 验证后的输出` |
| ← | 回滚，反转 | `迁移 ← 回滚` |
| ⇄ | 双向 | `同步 ⇄ 远程` |
| & | 并且，合并 | `🛡️ 安全 & ⚡ 性能` |
| \| | 分隔符，或者 | `react\|vue\|angular` |
| : | 定义，指明 | `范围: 文件\|模块` |
| » | 序列，然后 | `构建 » 测试 » 部署` |
| ∴ | 所以 | `测试 ❌ ∴ 代码损坏` |
| ∵ | 因为 | `慢 ∵ O(n²) 复杂度算法` |

### 状态与进度
| 符号 | 含义 | 用法 |
|--------|---------|-------|
| ✅ | 已完成，通过 | 任务成功完成 |
| ❌ | 失败，错误 | 需要立即关注 |
| ⚠️ | 警告 | 需要审查 |
| 🔄 | 进行中 | 当前活跃项 |
| ⏳ | 等待中，待处理 | 计划稍后进行 |
| 🚨 | 关键，紧急 | 高优先级行动 |

### 技术领域
| 符号 | 领域 | 用法 |
|--------|---------|-------|
| ⚡ | 性能 | 速度、优化 |
| 🔍 | 分析 | 搜索、调查 |
| 🔧 | 配置 | 设置、工具 |
| 🛡️ | 安全 | 保护、安全 |
| 📦 | 部署 | 打包、捆绑 |
| 🎨 | 设计 | UI、前端 |
| 🏗️ | 架构 | 系统结构 |

## 缩写系统

### 系统与架构
`cfg` 配置 (config) • `impl` 实施 (implementation) • `arch` 架构 (architecture) • `perf` 性能 (performance) • `ops` 操作 (operations) • `env` 环境 (environment)

### 开发流程
`req` 需求 (requirements) • `deps` 依赖 (dependencies) • `val` 验证 (validation) • `test` 测试 (testing) • `docs` 文档 (documentation) • `std` 标准 (standards)

### 质量与分析
`qual` 质量 (quality) • `sec` 安全 (security) • `err` 错误 (error) • `rec` 恢复 (recovery) • `sev` 严重性 (severity) • `opt` 优化 (optimization)

## 示例
```
标准模式：“身份验证系统的用户验证函数中存在安全漏洞”
Token 效率模式：“auth.js:45 → 🛡️ sec 风险 @ 用户 val()”

标准模式：“构建过程已成功完成，现在正在运行测试，然后进行部署”
Token 效率模式：“构建 ✅ » 测试 🔄 » 部署 ⏳”

标准模式：“性能分析显示由于 O(n²) 复杂度，算法运行缓慢”
Token 效率模式：“⚡ perf 分析：慢 ∵ O(n²) 复杂度”
```
