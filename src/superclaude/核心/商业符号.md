# BUSINESS_SYMBOLS.md - 商业分析符号系统

专为商业专家面板 (Business Panel) 分析设计的增强型符号系统，专注于战略重点和效率优化。

## 商业专用符号

### 战略分析
| 符号 | 含义 | 使用场景 |
|--------|---------|---------------|
| 🎯 | 战略目标、目的 | 关键目标和预期成果 |
| 📈 | 增长机会、趋势向好 | 市场增长、收入增加 |
| 📉 | 下降、风险、趋势向差 | 市场萎缩、威胁因素 |
| 💰 | 财务影响、收入 | 经济驱动因素、利润中心 |
| ⚖️ | 权衡、平衡 | 战略决策、资源分配 |
| 🏆 | 竞争优势 | 独特价值主张、强项 |
| 🔄 | 商业周期、反馈回路 | 循环模式、系统动力学 |
| 🌊 | 蓝海、新市场 | 开辟无人竞争的市场空间 |
| 🏭 | 行业、市场结构 | 竞争格局 |
| 🎪 | 卓越、紫牛 (Purple Cow) | 出众的产品、病毒式传播潜力 |

### 框架集成 (Expert Frameworks)
| 符号 | 专家 | 框架元素 |
|--------|--------|-------------------|
| 🔨 | Christensen | 待完成任务 (JTBD) |
| ⚔️ | Porter | 五力模型 |
| 🎪 | Godin | 紫牛/卓越产品 |

| 🚀 | Collins | 飞轮效应 |

### 分析过程
| 符号 | 过程阶段 | 描述 |
|--------|---------------|-------------|
| 🔍 | 调查 (Investigation) | 初步分析与探索 |
| 💡 | 洞察 (Insight) | 关键认知与突破 |
| 🤝 | 共识 (Consensus) | 专家达成一致的领域 |
| ⚡ | 张力 (Tension) | 富有成效的分歧 |
| 🎭 | 辩论 (Debate) | 对抗性分析模式 |
| ❓ | 苏格拉底式 (Socratic) | 问题驱动的探索 |
| 🧩 | 综合 (Synthesis) | 跨框架集成 |
| 📋 | 结论 (Conclusion) | 最终建议 |

### 商业逻辑流
| 符号 | 含义 | 商业上下文 |
|--------|---------|------------------|
| → | 导致、指向 | 市场趋势 → 机会 |
| ⇒ | 战略转型 | 当前状态 ⇒ 理想未来 |
| ← | 约束、限制 | 资源限制 ← 预算 |
| ⇄ | 相互影响 | 客户需求 ⇄ 产品开发 |
| ∴ | 战略结论 | 市场分析 ∴ 市场进入战略 |
| ∵ | 商业逻辑/理由 | 扩张 ∵ 市场机遇 |
| ≡ | 战略等效 | 策略 A ≡ 策略 B 的结果 |
| ≠ | 竞争差异化 | 我们的方案 ≠ 竞争对手 |

## Token 效率集成

### 压缩策略
- **专家声音压缩**: 在减少冗余的同时保持声音的真实性。
- **框架符号替换**: 对常见的框架概念使用符号。
- **结构化输出**: 使用组织良好的模板减少重复文本。
- **智能缩写**: 使用行业特定缩写并保留上下文。

### 商业缩写

### 性能优化
- **Token 预算**: 全面分析控制在 15-30K tokens。
- **专家缓存**: 存储专家人格 (Personas) 以供会话内重用。
- **框架重用**: 对相似内容缓存框架应用结果。
- **综合模板**: 预设结构化输出格式以提高效率。
- **并行分析**: 在可能的情况下，并行运行专家分析。

## 质量保证

### 真实性验证
- **声音一致性**: 每位专家保持其独特的沟通风格。
- **框架保真度**: 分析遵循真实的框架方法论。
- **交互现实主义**: 专家间的互动反映真实的专业动态。
- **综合完整性**: 合并后的洞察仍保留各框架的独立价值。

### 商业分析标准
- **战略相关性**: 分析解决真实的商业战略关切。
- **实施可行性**: 建议具有可操作性和现实性。
- **证据基础**: 结论由框架逻辑和商业证据支撑。
- **专业水平**: 分析符合高级管理层的商业沟通标准。
