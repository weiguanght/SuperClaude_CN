# Playwright MCP 服务器 (Playwright MCP Server)

**用途**：浏览器自动化以及具备真实浏览器交互能力的端到端 (E2E) 测试。

## 触发场景
- 浏览器测试及端到端 (E2E) 测试场景。
- 视觉测试、屏幕截图或 UI 验证请求。
- 表单提交及用户交互测试。
- 跨浏览器兼容性验证。
- 需要真实浏览器渲染的性能测试。
- 自动化 WCAG 合规性的无障碍测试。

## 何时选择
- **执行真实浏览器交互时**：当您需要实际的渲染效果而非仅仅是代码时。
- **优于单元测试**：进行集成测试、用户路径模拟、视觉验证。
- **针对 E2E 场景**：登录流程、表单提交、多页面工作流。
- **针对视觉测试**：屏显对比、响应式设计验证。
- **不适用于代码分析**：静态代码审查、语法检查、逻辑验证。

## 最佳协同组合
- **Sequential (顺序推理)**：由 Sequential 规划测试策略 → 由 Playwright 执行浏览器自动化。
- **Magic**：由 Magic 创建 UI 组件 → 由 Playwright 验证无障碍性与行为逻辑。

## 示例
```
"测试登录流程" → Playwright (浏览器自动化)
"检查表单验证是否生效" → Playwright (真实用户交互)
"截取响应式设计的屏幕快照" → Playwright (视觉测试)
"验证无障碍合规性" → Playwright (自动化 WCAG 测试)
"审查此函数的逻辑" → 原生 Claude (静态分析)
"解释身份验证代码" → 原生 Claude (代码审查)
```
