# Sequential MCP 服务器 (Sequential MCP Server)

**用途**：用于复杂分析和系统化解决问题的多步推理引擎。

## 触发场景
- 涉及多个层级的复杂调试场景。
- 架构分析与系统设计类问题。
- 使用 `--think`, `--think-hard`, `--ultrathink` 标志。
- 需要进行假设检验与验证的问题。
- 多组件故障调查。
- 需要系统性方法识别性能瓶颈。

## 何时选择
- **优于原生推理**：当问题涉及 3 个或更多相互关联的组件时。
- **针对系统分析**：根本原因分析 (RCA)、架构审查、安全评估。
- **当结构至关重要时**：对问题进行分解并收集证据能带来显著收益。
- **针对跨域问题**：涵盖前端、后端、数据库及基础设施的综合性问题。
- **不适用于简单任务**：基础解释、单文件变更、直接的修复。

## 最佳协同组合
- **Context7**：由 Sequential 协调分析 → 由 Context7 提供官方模式支持。
- **Magic**：由 Sequential 分析 UI 逻辑 → 由 Magic 实施结构化组件。
- **Playwright**：由 Sequential 确定测试策略 → 由 Playwright 执行验证。

## 示例
```
"为什么这个 API 这么慢？" → Sequential (系统化性能分析)
"设计一个微服务架构" → Sequential (结构化系统设计)
"调试此身份验证流程" → Sequential (多组件调查)
"分析安全漏洞" → Sequential (全方位的威胁建模)
"解释此函数" → 原生 Claude (基础解释)
"修复此拼写错误" → 原生 Claude (简单的变更)
```
