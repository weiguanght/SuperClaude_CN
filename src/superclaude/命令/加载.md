# /sc:load - 项目上下文加载

## 触发场景
- 会话初始化与项目上下文加载请求。
- 跨会话持久化与记忆检索需求。
- 项目激活与上下文管理要求。
- 会话生命周期管理与检查点 (Checkpoint) 加载场景。

## 用法
```
/sc:load [目标] [--type project|config|deps|checkpoint] [--refresh] [--analyze]
```

## 行为流程
1. **初始化 (Initialize)**：建立 Serena MCP 连接及会话上下文管理。
2. **探索 (Discover)**：分析项目结构并识别上下文加载需求。
3. **加载 (Load)**：检索项目记忆、检查点及跨会话持久化数据。
4. **激活 (Activate)**：建立项目上下文并为开发工作流做好准备。
5. **验证 (Validate)**：确保加载内容的完整性及会话就绪状态。

核心行为：
- 集成 Serena MCP 负责记忆管理与跨会话持久化。
- 具备全面上下文加载与验证的项目激活。
- 性能关键型操作，初始化的目标响应时间 <500ms。
- 基于检查点与记忆协调的会话生命周期管理。

## MCP 集成
- **Serena MCP**：强制集成的服务器，负责项目激活、记忆检索及会话管理。
- **记忆操作**：跨会话持久化、检查点加载及上下文恢复。
- **性能指标**：核心操作 <200ms，创建检查点 <1s。

## 工具协作
- **activate_project**：核心项目激活与上下文建立。
- **list_memories/read_memory**：记忆检索与会话上下文加载。
- **Read/Grep/Glob**：项目结构分析与配置发现。
- **Write**：会话上下文记录与检查点创建。

## 关键模式
- **项目激活**：目录分析 → 记忆检索 → 上下文建立。
- **会话恢复**：检查点加载 → 上下文验证 → 工作流准备。
- **记忆管理**：跨会话持久化 → 上下文连续性 → 开发效率提升。
- **性能优先**：快速初始化 → 立即产出 → 会话就绪。

## 示例

### 基础项目加载
```
/sc:load
```
# 加载当前目录的项目上下文，并集成 Serena 记忆
# 建立会话上下文并为开发工作流做好准备

### 特定项目加载
```
/sc:load /path/to/project --type project --analyze
```
# 对特定项目进行综合分析并加载
# 激活项目上下文并检索跨会话记忆

### 检查点恢复
```
/sc:load --type checkpoint --checkpoint session_123
```
# 恢复特定的检查点及其会话上下文
# 在完整保留上下文的情况下继续此前的会话工作

### 依赖项上下文加载
```
/sc:load --type deps --refresh
```
# 通过全新分析加载依赖项上下文
# 更新项目理解及依赖映射

## 边界

**能力范围：**
- 利用 Serena MCP 集成进行记忆管理并加载项目上下文。
- 提供具备跨会话持久化能力的会话生命周期管理。
- 建立具备全面上下文加载的项目激活。

**非能力范围：**
- 在没有获得明确许可的情况下修改项目结构或配置。
- 在没有适当的 Serena MCP 集成与验证的情况下加载上下文。
- 在未能保存检查点的情况下覆盖现有的会话上下文。
