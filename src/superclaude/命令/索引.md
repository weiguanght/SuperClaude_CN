# /sc:index - 项目文档生成

## 触发场景
- 项目文档的创建与维护需求。
- 知识库的生成与组织整理。
- API 文档以及结构分析要求。
- 交叉引用与导航增强请求。

## 用法
```
/sc:index [目标] [--type docs|api|structure|readme] [--format md|json|yaml]
```

## 行为流程
1. **分析 (Analyze)**：检查项目结构并识别关键文档组件。
2. **组织 (Organize)**：应用智能组织模式及交叉引用策略。
3. **生成 (Generate)**：应用框架特定的模式创建全面文档。
4. **验证 (Validate)**：确保文档的完整性并符合质量标准。
5. **维护 (Maintain)**：更新现有文档，同时保留手动添加的内容和自定义设置。

核心行为：
- 根据文档范围和复杂度协调多个人格 (架构师、文案专家、质量工程师)。
- 通过 Sequential MCP 进行系统化分析，执行综合性的文档生成工作流。
- 通过 Context7 MCP 集成框架特定的模式与文档标准。
- 具备交叉引用能力和自动化维护机制的智能组织。

## MCP 集成
- **Sequential MCP**：集成复杂的多步项目分析与系统化文档生成。
- **Context7 MCP**：提供框架特定的文档模式及既定标准。
- **人格协作**：架构师 (结构)、文案专家 (内容)、质量工程师 (验证)。

## 工具协作
- **Read/Grep/Glob**：项目结构分析与内容提取，为文档生成提供支持。
- **Write**：具备智能组织与交叉引用能力的文档创建。
- **TodoWrite**：针对复杂多组件文档工作流的进度追踪。
- **Task**：针对需要系统化协调的大规模文档编制进行高级委派。

## 关键模式
- **结构分析**：项目检查 → 组件识别 → 逻辑组织 → 交叉引用。
- **文档类型**：API 文档 → 结构文档 → README → 知识库方法。
- **质量验证**：完整性评估 → 准确性核实 → 标准遵循情况检查 → 维护计划。
- **框架集成**：参考 Context7 模式 → 遵循官方标准 → 应用最佳实践 → 一致性验证。

## 边界

**能力范围：**
- 生成具备智能组织与交叉引用的全面项目文档。
- 应用多个人格协作进行系统化分析与质量验证。
- 提供框架特定的模式及既定文档标准。

**非能力范围：**
- 在没有获得明确更新权限的情况下覆盖现有的手动文档。
- 在缺乏适当的项目结构分析与验证的情况下生成文档。
- 绕过既定的文档标准或质量要求。
