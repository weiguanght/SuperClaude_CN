# /sc:git - Git 操作

## 触发场景
- Git 仓库常规操作：status, add, commit, push, pull, branch。
- 需要智能生成提交信息 (Commit Message)。
- 仓库工作流优化请求。
- 分支管理与合并操作。

## 用法
```
/sc:git [操作名称] [参数] [--smart-commit] [--interactive]
```

## 行为流程
1. **分析 (Analyze)**：检查仓库状态及工作目录变更。
2. **验证 (Validate)**：确保操作适配当前的 Git 上下文。
3. **执行 (Execute)**：通过智能自动化运行 Git 命令。
4. **优化 (Optimize)**：应用智能提交信息和工作流模式。
5. **汇报 (Report)**：提供状态更新及后续步骤指南。

核心行为：
- 基于变更分析生成规范提交 (Conventional Commit) 信息。
- 应用一致的分支命名规范。
- 引导式的合并冲突解决处理。
- 提供清晰的状态摘要和工作流建议。

## 工具协作
- **Bash**：Git 命令执行与仓库操作。
- **Read**：仓库状态分析与配置审查。
- **Grep**：日志解析与状态分析。
- **Write**：提交信息生成与文档编写。

## 关键模式
- **智能提交 (Smart Commits)**：分析变更 → 生成规范提交信息。
- **状态分析**：仓库状态 → 可操作的建议。
- **分支策略**：强制执行一致的命名与工作流。
- **错误恢复**：冲突解决与状态恢复指南。

## 示例

### 智能状态分析
```
/sc:git status
```
# 分拆仓库状态并提供变更摘要
# 提供后续步骤及工作流建议

### 智能提交
```
/sc:git commit --smart-commit
```
# 根据变更分析生成规范提交信息
# 应用最佳实践和一致的格式编码

### 交互式操作
```
/sc:git merge feature-branch --interactive
```
# 引导式合并，并提供冲突解决协助

## 边界

**能力范围：**
- 通过智能自动化执行 Git 操作。
- 根据变更分析生成规范提交信息。
- 提供工作流优化和最佳实践指南。

**非能力范围：**
- 在未获得明确授权的情况下修改仓库配置。
- 在未经确认的情况下执行破坏性操作。
- 处理需要深度人工干预的复杂合并。
