# /sc:task - 增强型任务管理

## 触发场景
- 需要多智能体协作与委派的复杂任务。
- 需要结构化工作流管理与跨会话持久化的项目。
- 需要智能 MCP 服务器路由及领域专业知识的操作。
- 可从系统化执行及渐进式增强中获益的任务。

## 用法
```
/sc:task [行动] [目标] [--strategy systematic|agile|enterprise] [--parallel] [--delegate]
```

## 行为流程
1. **分析 (Analyze)**：解析任务需求并确定最优执行策略。
2. **委派 (Delegate)**：路由至合适的 MCP 服务器并激活相关的人格。
3. **协调 (Coordinate)**：通过智能工作流管理与并行处理执行任务。
4. **验证 (Validate)**：应用质量门禁进行全面的任务完成度校验。
5. **优化 (Optimize)**：分析性能并提供增强建议。

核心行为：
- 跨架构师、前端、后端、安全、DevOps 领域的多人格协作。
- 智能化的 MCP 服务器路由 (Sequential, Context7, Magic, Playwright, Morphllm, Serena)。
- 具备渐进式任务增强与跨会话持久化的系统化执行。
- 具备层级化拆解与依赖管理的高级任务委派。

## MCP 集成
- **Sequential MCP**：负责复杂的多步任务分析与系统化执行规划。
- **Context7 MCP**：提供框架特定的模式与实施最佳实践。
- **Magic MCP**：协调 UI/UX 任务并集成设计系统。
- **Playwright MCP**：集成测试工作流并自动化验证过程。
- **Morphllm MCP**：负责大规模任务转换及基于模式的优化。
- **Serena MCP**：负责跨会话任务持久化与项目记忆管理。

## 工具协作
- **TodoWrite**：实现 Epic (史诗) → Story (故事) → Task (任务) 级别的层级化任务拆解与进度追踪。
- **Task**：针对复杂多智能体协作与子任务管理的高级委派。
- **Read/Write/Edit**：任务文档记录与实施协调。
- **sequentialthinking**：为复杂的任务依赖分析提供结构化推理。

## 关键模式
- **任务层级**：Epic 级目标 → Story 协调 → Task 执行 → Subtask 粒度。
- **策略选择**：系统化 (全面执行) → 敏捷 (迭代执行) → 企业级 (治理执行)。
- **多智能体协作**：人格激活 → MCP 路由 → 并行执行 → 结果集成。
- **跨会话管理**：任务持久化 → 上下文连续性 → 渐进式增强。

## 示例

### 复杂功能开发
```
/sc:task create "企业级身份验证系统" --strategy systematic --parallel
```
# 具备多领域协作的全面任务拆解
# 激活架构师、安全、后端、前端人格

### 敏捷 Sprint 协调
```
/sc:task execute "功能待办列表 (Backlog)" --strategy agile --delegate
```
# 具备智能委派的迭代式任务执行
# 用于 Sprint 连续性的跨会话持久化

### 多领域集成
```
/sc:task execute "微服务平台" --strategy enterprise --parallel
```
# 具备合规验证的企业级规模协调
# 跨多个技术领域的并行执行

## 边界

**能力范围：**
- 通过多智能体协作与智能委派执行复杂任务。
- 提供具备跨会话持久化的层级化任务拆解。
- 协调多个 MCP 服务器与人格，以实现最优的任务产出。

**非能力范围：**
- 执行无需高级编排的简单任务。
- 为了速度或便利性而降低质量标准。
- 在缺乏适当验证与质量门禁的情况下运行。
