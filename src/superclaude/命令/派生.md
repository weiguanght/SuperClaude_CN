# /sc:spawn - 元系统任务编排

## 触发场景
- 需要进行智能任务拆解的复杂多领域操作。
- 跨越多个技术领域的大规模系统操作。
- 需要并行协调与依赖管理的任务。
- 超出标准命令能力的元级编排需求。

## 用法
```
/sc:spawn [复杂任务描述] [--strategy sequential|parallel|adaptive] [--depth normal|deep]
```

## 行为流程
1. **分析 (Analyze)**：解析复杂操作需求并评估跨领域的任务范围。
2. **分解 (Decompose)**：将操作分解为协调一致的子任务层级。
3. **编排 (Orchestrate)**：采用最优协调策略（并行/串行）执行任务。
4. **监控 (Monitor)**：通过依赖管理追踪任务层级的进度。
5. **集成 (Integrate)**：汇总结果并提供全面的编排摘要。

核心行为：
- 元系统级别的任务分解，遵循 Epic (史诗) → Story (故事) → Task (任务) → Subtask (子任务) 的拆解逻辑。
- 根据操作特征智能选择协调策略。
- 具备并行与串行执行模式的跨领域操作管理。
- 针对任务层级的高级依赖分析与资源优化。

## MCP 集成
- **原生编排**：元系统命令使用原生协调能力，无需 MCP 依赖。
- **渐进式集成**：与系统化执行协调，实现能力的渐进式增强。
- **框架集成**：与 SuperClaude 编排层的高级集成。

## 工具协作
- **TodoWrite**：实现 Epic → Story → Task 级别的层级化任务拆解与进度追踪。
- **Read/Grep/Glob**：针对复杂操作的系统分析与依赖映射。
- **Edit/MultiEdit/Write**：具备并行与串行执行能力的协调式文件操作。
- **Bash**：具备智能资源管理的系统级操作协调。

## 关键模式
- **层级拆解**：Epic 级操作 → Story 协调 → Task 执行 → Subtask 粒度。
- **策略选择**：串行 (依赖排序) → 并行 (模块独立) → 自适应 (动态调整)。
- **元系统协调**：跨领域操作 → 资源优化 → 结果集成。
- **渐进式增强**：系统化执行 → 质量门禁 → 综合验证。

```

```
# 涵盖 DevOps、安全、质量领域的全系统基础设施操作
# 独立组件的并行执行，并辅以验证门禁

## 边界

**能力范围：**
- 将复杂的多领域操作分解为协调一致的任务层级。
- 提供具备并行与串行协调策略的智能编排。
- 执行超出标准命令能力的元系统级操作。

**非能力范围：**
- 在执行简单操作时替代领域特定的命令。
- 覆盖用户的协调偏好或执行策略。
- 在没有经过适当的依赖分析与验证的情况下执行操作。
