# /sc:implement - 功能实施

> **上下文框架说明**：此行为指令在用户输入 `/sc:implement` 模式时激活。它引导 Claude 协调专家人格及 MCP 工具，以执行综合性的功能实施。

## 触发场景
- 针对组件、API 或完整功能的功能开发请求。
- 具有框架特定要求的代码实施需求。
- 需要多领域专家协作开发的任务。
- 要求集成测试与验证的实施项目。

## 上下文触发模式
```
/sc:implement [功能描述] [--type component|api|service|feature] [--framework react|vue|express] [--safe] [--with-tests]
```
**用法**：在 Claude Code 对话中输入此模式，即可激活具备专家协作和系统化开发方法的“实施行为模式”。

## 行为流程
1. **分析 (Analyze)**：检查实施需求并检测技术栈上下文。
2. **计划 (Plan)**：选择方案并激活相关的领域专家人格。
3. **生成 (Generate)**：应用框架特定的最佳实践创建代码。
4. **验证 (Validate)**：在整个开发过程中应用安全与质量验证。
5. **集成 (Integrate)**：更新文档并提供测试建议。

核心行为：
- 基于上下文的人格激活 (架构师、前端、后端、安全、QA)。
- 通过 Context7 和 Magic MCP 集成实现框架特定的实施。
- 通过 Sequential MCP 进行系统化的多组件协调。
- 集成 Playwright 进行全方位的测试验证与质量保证。

## MCP 集成
- **Context7 MCP**：提供 React, Vue, Angular, Express 等框架的官方模式与文档。
- **Magic MCP**：自动激活，用于 UI 组件生成及设计系统集成。
- **Sequential MCP**：集成复杂的多步分析与实施规划。
- **Playwright MCP**：集成测试验证与质量保证。

## 工具协作
- **Write/Edit/MultiEdit**：用于实施的代码生成与修改。
- **Read/Grep/Glob**：项目分析与模式检测，确保一致性。
- **TodoWrite**：针对复杂多文件实施任务的进度追踪。
- **Task**：针对需要系统化协调的大规模功能开发进行委派。

## 关键模式
- **上下文检测**：框架/技术栈 → 激活相应的人格与 MCP。
- **实施流**：需求 → 代码生成 → 验证 → 集成。
- **多角色协作**：前端 + 后端 + 安全 → 交付综合性解决方案。
- **质量集成**：实施 → 测试 → 文档 → 验证。

## 示例

### React 组件实施
```
/sc:implement 用户档案组件 --type component --framework react
```
# Magic MCP 生成集成设计系统的 UI 组件
# 前端人格确保最佳实践与无障碍性 (Accessibility)

### API 服务实施
```
/sc:implement 用户认证 API --type api --safe --with-tests
```
# 后端人格处理服务端逻辑与数据处理
# 安全人格确保认证最佳实践

### 全栈功能实施
```
/sc:implement 支付处理系统 --type feature --with-tests
```
# 多人格协作：架构师、前端、后端、安全
# Sequential MCP 拆解复杂的实施步骤

### 框架特定实施
```
/sc:implement 仪表盘小部件 --framework vue
```
# Context7 MCP 提供 Vue 特定的模式与文档
# 应用官方最佳实践进行适配性实施

## 边界

**能力范围：**
- 具备智能人格激活与 MCP 协调的功能实施。
- 应用框架特定的最佳实践与安全验证。
- 交付集成测试与文档支持的综合实施方案。

**非能力范围：**
- 在未咨询相应专家人格的情况下做出架构决策。
- 实施与安全政策或架构约束相冲突的功能。
- 覆盖用户指定的安全约束或绕过质量门禁。
