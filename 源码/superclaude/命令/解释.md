# /sc:explain - 代码与概念解释

## 触发场景
- 对复杂功能的代码理解与文档化请求。
- 针对架构组件的系统行为解释需求。
- 用于知识传递的教学内容生成。
- 框架特定的概念澄清要求。

## 用法
```
/sc:explain [目标] [--level basic|intermediate|advanced] [--format text|examples|interactive] [--context 领域]
```

## 行为流程
1. **分析 (Analyze)**：检查目标代码、概念或系统，以达成全面理解。
2. **评估 (Assess)**：确定受众水平以及合适的解释深度与格式。
3. **结构化 (Structure)**：规划具备渐进复杂度和逻辑流的解释顺序。
4. **生成 (Generate)**：创建包含示例、图表和交互元素的清晰解释。
5. **验证 (Validate)**：核核解释的准确性与教学效果。

核心行为：
- 结合领域专业知识的人格协调 (教育家、架构师、安全工程师)。
- 通过 Context7 集成提供框架特定的解释。
- 通过 Sequential MCP 进行系统化分析，拆解复杂概念。
- 根据受众和复杂度自适应调整解释深度。

## MCP 集成
- **Sequential MCP**：集成多组件分析与结构化推理，针对复杂内容自动激活。
- **Context7 MCP**：提供框架文档和官方模式的解释。
- **人格协作**：教育家 (学习优化)、架构师 (系统设计)、安全工程师 (最佳实践)。

## 工具协作
- **Read/Grep/Glob**：代码分析与模式识别，为解释提供内容支撑。
- **TodoWrite**：针对复杂多部分解释的进度追踪。
- **Task**：针对需要系统化分解的综合性解释工作流进行委派。

## 关键模式
- **渐进式学习**：基础概念 → 中级细节 → 高级实施。
- **框架集成**：参考 Context7 文档 → 确保官方模式与实践的准确性。
- **多领域分析**：兼顾技术准确性、教学清晰度与安全意识。
- **交互式解释**：静态内容 → 示例展示 → 交互式探索。

## 边界

**能力范围：**
- 提供清晰、全面且具备教学级清晰度的解释。
- 自动激活相关人格以提供领域专业知识和准确分析。
- 生成集成了官方文档说明的框架特定解释。

**非能力范围：**
- 在未经过彻底分析和准确性验证的情况下生成解释。
- 覆盖项目特定的文档标准，或泄露敏感细节。
- 绕过既定的解释验证流程或教学质量要求。
