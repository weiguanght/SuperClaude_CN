# /sc:research - 深度调研命令

> **上下文框架说明**：此命令激活具备自适应规划、多跳推理及基于证据的综合能力的全面调研功能。

## 触发场景
- 超出知识截止日期的调研问题。
- 复杂的调研问题。
- 时事与实时信息查询。
- 学术或技术调研需求。
- 市场分析与竞争情报。

## 上下文触发模式
```
/sc:research "[查询内容]" [--depth quick|standard|deep|exhaustive] [--strategy planning|intent|unified]
```

## 行为流程

### 1. 理解 (Understanding - 5-10% 权重)
- 评估查询的复杂度与模糊性。
- 识别所需的信息类型。
- 确定资源需求。
- 定义成功标准。

### 2. 规划 (Planning - 10-15% 权重)
- 根据复杂度选择规划策略。
- 识别并行化的机会。
- 对调研问题进行拆解。
- 创建调研里程碑。

### 3. 任务化 (TodoWrite - 5% 权重)
- 创建自适应的任务层级。
- 根据查询复杂度缩放任务规模（3-15 个任务）。
- 建立任务依赖关系。
- 设置进度追踪。

### 4. 执行 (Execution - 50-60% 权重)
- **并行优先搜索**：始终对相似查询进行批处理。
- **智能提取**：根据内容复杂度进行路由。
- **多跳探索 (Multi-hop)**：跟踪实体与概念链。
- **证据收集**：追踪来源与置信度。

### 5. 追踪 (Tracking - 持续进行)
- 监控 TodoWrite 进度。
- 更新置信度分值。
- 记录成功模式。
- 识别信息差距。

### 6. 验证 (Validation - 10-15% 权重)
- 验证证据链。
- 检查来源可信度。
- 解决矛盾点。
- 确保完整性。

## 关键模式

### 并行执行
- 对所有独立的搜索进行批处理。
- 执行并发提取。
- 仅在有依赖关系时使用串行执行。

### 证据管理
- 追踪搜索结果。
- 在可用时提供清晰的引用。
- 显式标注不确定性。

### 自适应深度
- **快速 (Quick)**：基础搜索，1 跳，摘要输出。
- **标准 (Standard)**：扩展搜索，2-3 跳，结构化报告。
- **深度 (Deep)**：全面搜索，3-4 跳，详细分析。
- **穷举 (Exhaustive)**：最大深度，5 跳，完整调研。

## MCP 集成
- **Tavily**：主要的搜索与提取引擎。
- **Sequential**：复杂的推理与综合。
- **Playwright**：针对重度 JavaScript 内容的提取。
- **Serena**：调研会话的持久化。

## 输出标准
- 将报告保存至 `claudedocs/research_[主题]_[时间戳].md`。
- 包含执行摘要。
- 提供置信水平。
- 列出包含引用的所有来源。

## 示例
```bash
/sc:research "2024 年量子计算的最新进展"
/sc:research "AI 编码助手的竞争分析" --depth deep
/sc:research "分布式系统的最佳实践" --strategy unified
```

## 边界
**能力范围**：获取实时信息、执行智能搜索、提供基于证据的分析。
**非能力范围**：提供无来源的主张、跳过验证步骤、访问受限内容。
