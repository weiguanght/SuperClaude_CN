# /sc:analyze - 代码分析与质量评估

## 触发场景
- 对项目或特定组件的代码质量评估请求。
- 安全漏洞扫描与合规性验证需求。
- 性能瓶颈识别与优化规划。
- 架构审查与技术债评估要求。

## 用法
```
/sc:analyze [目标] [--focus quality|security|performance|architecture] [--depth quick|deep] [--format text|json|report]
```

## 行为流程
1. **探索 (Discover)**：利用语言检测和项目分析对源文件进行分类。
2. **扫描 (Scan)**：应用领域特定的分析技术和模式匹配。
3. **评估 (Evaluate)**：生成带有严重分级和影响评估的优先发现项。
4. **建议 (Recommend)**：创建带有实施指南的可操作建议。
5. **汇报 (Report)**：展示包含指标和改进路线图的综合分析。

核心行为：
- 结合静态分析与启发式评估的多领域分析。
- 智能文件发现与特定语言的模式识别。
- 基于严重程度的发现项与建议优先级排序。
- 包含指标、趋势和可操作见解的全面报告。

## 工具协作
- **Glob**：文件发现与项目结构分析。
- **Grep**：模式分析与代码搜索操作。
- **Read**：源代码检查与配置分析。
- **Bash**：外部分析工具执行与验证。
- **Write**：报告生成与指标记录。

## 关键模式
- **领域分析**：质量/安全/性能/架构 → 专业化评估。
- **模式识别**：语言检测 → 适配相应的分析技术。
- **严重性评估**：问题分类 → 优先建议。
- **报告生成**：分析结果 → 结构化文档。

## 示例

### 综合项目分析
```
/sc:analyze
# 对整个项目进行多领域分析
# 生成包含关键发现和路线图的综合报告
```

### 专项安全评估
```
/sc:analyze src/auth --focus security --depth deep
# 对身份验证组件进行深度安全分析
# 漏洞评估及详细的修复指南
```

### 性能优化分析
```
/sc:analyze --focus performance --format report
# 性能瓶颈识别
# 生成包含优化建议的报告
```

### 快速质量检查
```
/sc:analyze src/components --focus quality --depth quick
# 对组件目录进行快速质量评估
# 识别代码异味 (Code Smells) 和可维护性问题
```

## 边界

**能力范围：**
- 执行涵盖多个领域的全面静态代码分析。
- 生成带有严重性评分的发现项及可操作建议。
- 提供包含指标和改进指南的详细报告。

**非能力范围：**
- 执行需要代码编译或运行时环境的动态分析。
- 在未获得用户明确授权的情况下修改源代码或应用修复。
- 分析除导入和使用模式以外的外部依赖项。
