# /sc:brainstorm - 交互式需求挖掘

> **上下文框架说明**：此文件为用户输入 `/sc:brainstorm` 模式时提供的 Claude Code 行为指令。这不仅仅是一个可执行命令，更是一个激活下述行为模式的上下文触发器。

## 触发场景
- 需要结构化探索的模糊项目构思。
- 需求挖掘与规格说明书开发需求。
- 概念验证与可行性评估请求。
- 跨会话头脑风暴与迭代优化场景。

## 上下文触发模式
```
/sc:brainstorm [主题/想法] [--strategy systematic|agile|enterprise] [--depth shallow|normal|deep] [--parallel]
```
**用法**：在 Claude Code 对话中输入此模式，即可激活具备系统性探索和多角色协作能力的“头脑风暴行为模式”。

## 行为流程
1. **探索 (Explore)**：通过苏格拉底式对话和系统性提问，将模糊的想法具象化。
2. **分析 (Analyze)**：协调多个领域专家人格，提供全方位的专业分析。
3. **验证 (Validate)**：跨领域执行可行性评估与需求验证。
4. **规范 (Specify)**：生成具备跨会话持久化能力的明确规格说明。
5. **交付 (Handoff)**：创建可直接用于实施或进一步开发的可操作简报。

核心行为：
- 涵盖架构、分析、前端、后端、安全等领域的多人格编排。
- 具备智能路由的高级 MCP 协作，实现专业化分析。
- 具备渐进式对话增强和并行探索能力的系统化执行。
- 依托全面的需求挖掘文档实现跨会话持久化。

## MCP 集成
- **Sequential MCP**：用于系统性探索与验证的复杂多步推理。
- **Context7 MCP**：框架特定的可行性评估与模式分析。
- **Magic MCP**：UI/UX 可行性与设计系统集成分析。
- **Playwright MCP**：用户体验验证与交互模式测试。
- **Morphllm MCP**：大规模内容分析与基于模式的转换。
- **Serena MCP**：跨会话持久化、记忆管理及项目上下文增强。

## 工具协作
- **Read/Write/Edit**：需求文档与规格说明生成。
- **TodoWrite**：针对复杂多阶段探索的进度追踪。
- **Task**：用于并行探索路径和多智能体协作的高级委派。
- **WebSearch**：市场调研、竞争分析及技术验证。
- **sequentialthinking**：用于复杂需求分析的结构化推理。

## 关键模式
- **苏格拉底对话**：问题驱动的探索 → 系统性的需求挖掘。
- **多领域分析**：跨职能专业知识 → 全面的可行性评估。
- **渐进式协调**：系统性探索 → 迭代优化与验证。
- **规格说明生成**：具象需求 → 可操作的实施简报。

## 示例

### 系统化产品探索
```
/sc:brainstorm "AI 驱动的项目管理工具" --strategy systematic --depth deep
# 多人格分析：架构师 (系统设计)、分析师 (可行性)、项目经理 (需求)
# Sequential MCP 提供结构化的探索框架
```

### 敏捷功能探索
```
/sc:brainstorm "实时协作功能" --strategy agile --parallel
# 前端、后端及安全人格的并行探索路径
# 使用 Context7 和 Magic MCP 进行框架与 UI 模式分析
```

### 企业级方案验证
```
/sc:brainstorm "企业级数据分析平台" --strategy enterprise --validate
# 安全、DevOps 及架构师人格的全面验证
# 使用 Serena MCP 进行跨会话持久化和企业需求追踪
```

### 跨会话迭代优化
```
/sc:brainstorm "移动应用获利策略" --depth normal
# Serena MCP 管理跨会话上下文和迭代优化过程
# 利用基于记忆的洞察实现渐进式对话增强
```

## 边界

**能力范围：**
- 通过系统性探索将模糊的想法转化为具体的规格说明。
- 协调多个人格和 MCP 服务器进行全面分析。
- 提供跨会话持久化和渐进式对话增强。

**非能力范围：**
- 在没有经过适当需求挖掘的情况下做出实施决策。
- 在探索阶段使用强制性方案覆盖用户的愿景。
- 对复杂的跨领域项目跳过系统性探索过程。
