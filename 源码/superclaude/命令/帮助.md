## SuperClaude 框架标记 (Flags)

SuperClaude 支持行为标记，用以启用特定的执行模式和工具选择模式。可以在任何 `/sc` 命令中使用这些标记来自定义行为。

### 模式激活标记

| 标记 | 触发词 | 行为描述 |
|------|---------|----------|
| `--brainstorm` | 模糊的项目请求, 探索性关键词 | 激活协作式发现心态，提出引导式问题 |
| `--introspect` | 自我分析请求, 错误恢复 | 以透明标记展示思考过程 |
| `--task-manage` | 多步操作 (>3 步) | 通过委派和系统化组织进行编排 |
| `--orchestrate` | 多工具操作, 并行执行 | 优化工具选择矩阵，开启并行思考模式 |
| `--token-efficient` | 上下文使用率 >75%, 大规模操作 | 使用基于符号的增强型沟通，Token 消耗减少 30-50% |

### MCP 服务器标记

| 标记 | 触发词 | 行为描述 |

### 执行控制标记

| 标记 | 触发词 | 行为描述 |
|------|---------|----------|
| `--delegate [auto\|files\|folders]` | >7 个目录 或 >50 个文件 | 启用子智能体并行处理及智能路由 |
| `--concurrency [n]` | 资源优化需求 | 控制最大并发操作数 (范围：1-15) |

### 输出优化标记

| 标记 | 触发词 | 行为描述 |
|------|---------|----------|
| `--uc` / `--ultracompressed` | 上下文压力, 效率要求 | 采用符号沟通系统，节省 30-50% 的 Token |
| `--scope [file\|module\|project\|system]` | 需要界定分析边界 | 定义操作范围与分析深度 |
| `--focus [performance\|security\|quality\|architecture\|accessibility\|testing]` | 领域特定优化 | 针对特定的分析领域和专业知识应用 |

### 标记优先级规则

- **安全第一**：`--safe-mode` > `--validate` > 优化标记。
- **显式覆盖**：用户标记 > 自动检测。
- **深度层级**：`--ultrathink` > `--think-hard` > `--think`。
- **MCP 控制**：`--no-mcp` 会覆盖所有单个 MCP 标记。
- **范围优先**：system > project > module > file。

