# /sc:document - 专项文档生成

## 触发场景
- 对特定组件、函数或功能的文档请求。
- API 文档及参考资料的生成需求。
- 代码注释与行内文档的要求。
- 用户指南及技术文档的创建请求。

## 用法
```
/sc:document [目标] [--type inline|external|api|guide] [--style brief|detailed]
```

## 行为流程
1. **分析 (Analyze)**：检查目标组件的结构、接口及功能。
2. **识别 (Identify)**：确定文档需求以及目标受众。
3. **生成 (Generate)**：根据类型和风格创建相应的文档内容。
4. **格式化 (Format)**：应用一致的结构和组织模式。
5. **集成 (Integrate)**：确保与项目现有文档体系的兼容性。

核心行为：
- 代码结构分析，包括 API 提取与使用模式识别。
- 多格式文档生成（行内、外部、API 参考、指南）。
- 一致的格式设置与交叉引用集成。
- 遵循特定语言的文档模式与惯例。

## 工具协作
- **Read**：组件分析与现有文档审阅。
- **Grep**：引用提取与模式识别。
- **Write**：创建具有正确格式的文档文件。
- **Glob**：处理多文件文档项目及组织工作。

## 关键模式
- **行内文档**：代码分析 → JSDoc/Docstring 生成 → 行内注释。
- **API 文档**：接口提取 → 参考资料 → 使用示例。
- **用户指南**：功能分析 → 教程内容 → 实施指南。
- **外部文档**：组件概览 → 详细规格 → 集成说明。

## 示例

### 行内代码文档
```
/sc:document src/auth/login.js --type inline
```
# 生成包含参数和返回值描述的 JSDoc 注释
# 为函数和类添加全面的行内文档

### API 参考生成
```
/sc:document src/api --type api --style detailed
```
# 创建包含端点和模式 (Schema) 的综合 API 文档
# 生成使用示例与集成准则

### 用户指南创建
```
/sc:document payment-module --type guide --style brief
```
# 创建面向用户的文档，并附带实用示例
# 侧重于实施模式与常见用例

### 组件文档
```
/sc:document components/ --type external
```
# 为组件库生成外部文档文件
# 包含 Props、使用示例及集成模式

## 边界

**能力范围：**
- 为特定组件和功能生成针对性的文档。
- 根据目标受众的需求创建多种文档格式。
- 与现有文档体系集成并保持一致性。

**非能力范围：**
- 在没有经过适当的代码分析和上下文理解的情况下生成文档。
- 覆盖现有的文档标准或项目特定的惯例。
- 创建会泄露敏感实施细节的文档。
