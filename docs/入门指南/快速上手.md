# 🚀 SuperClaude 快速上手指南

### **专为 Claude Code 打造的上下文工程框架**

> **💡 核心理念**：SuperClaude **并不替换** Claude Code，而是通过注入特定的行为上下文，来**配置并增强**它的原生能力。

---

## 🎯 **工作原理**
当您输入 `/sc:brainstorm` 时，Claude Code 会读取预设在 `.md` 文件中的行为指令。这意味着 AI 不是在“瞎聊”，而是在执行一套经过调优的专业开发流程。

---

## ⚡ **即刻开始**

### **第一步：安装 (命令行)**
```bash
# 推荐方式
pipx install SuperClaude && SuperClaude install
```

### **第二步：使用 (Claude Code)**
```text
/sc:brainstorm "我想做个任务管理应用"  # 开启需求调研
/sc:analyze src/                     # 深度分析源码
/sc:implement "用户注册功能"           # 按照最佳实践编写代码
@agent-security "检查鉴权逻辑"         # 委派安全专家进行审计
```

---

## 🔧 **四大核心支柱**
1. **指令集 (Commands)**：21 条 `/sc:` 命令，自动化繁琐的开发任务。
2. **专家团 (Agents)**：14 位领域专家（如架构师、安全工程师），提供专业见解。
3. **行为模式 (Modes)**：6 种针对性模式，让 AI 的思考更具深度。
4. **工具扩展 (MCP)**：6 处服务器集成，扩展 AI 的工具箱（如文档查阅、UI 设计）。

---

## 🌟 **典型开发周期**
1. **调研** (`/sc:brainstorm`)：挖掘需求。
2. **分析** (`/sc:analyze`)：理清现有架构。
3. **规划** (`/sc:workflow`)：制定实施路线图。
4. **实施** (`/sc:implement`)：编写工业级代码。
5. **验证** (`/sc:test`)：完成自动化测试。
6. **保存** (`/sc:save`)：存档当前进度，以便下次继续。

---

## 🎯 **何时使用 SuperClaude？**
- **适合**：从零构建完整项目、系统化重构旧代码、需要领域专家把关、跨模块的复杂开发任务。
- **不适合**：简单的算法问答、快速修复一两行 Bug、或是仅仅想让 AI 解释某个基础概念（标准 Claude Code 足矣）。
