# SuperClaude 验证与调试指南 (Troubleshooting)

由于 SuperClaude 是由一系列 Markdown 指令构成的框架，所谓的“验证”重点在于确保文件被正确放置、内容可读且能被 Claude Code 有效触发。

## 安装核验流程

### 1. 基础状态检查
- 运行 `python3 -m SuperClaude --version` 检查安装工具版本。
- 确认 `claude --version` 工作正常。
- 检查本地是否存在 `~/.claude/` 目录及其核心文件（`CLAUDE.md`, `agents/`, `commands/` 等）。

### 2. 组件完整性统计
- **专家 (Agents)**：应有 14-20 个文件。
- **命令 (Commands)**：应有 21-30 个文件。
- **模式 (Modes)**：应有 5-7 个文件。

## 故障诊断

### 情况 A：命令无法触发
- **对策**：检查 `~/.claude/commands/` 下是否存在对应的 `.md` 文件，并查看文件头部是否包含正确的触发描述。必要时使用 `python3 -m setup install --components commands --force` 重新修复。

### 情况 B：专家不响应调用
- **对策**：核实调用模式是否正确（如 `@agent-security`）。检查 `CLAUDE.md` 中的 `@import` 路径是否正确。

### 情况 C：MCP 服务器失效
- **对策**：核实 `node --version`（应为 16+）。检查 `~/.claude.json` 中的配置是否包含有效的 JSON 语法及路径。

## 常用修复脚本

### 一键修复文件权限
```bash
chmod 644 ~/.claude/*.md
chmod 755 ~/.claude ~/.claude/agents ~/.claude/commands
```

### 环境自动诊断 (示例)
您可以编写一个简单的脚本，检查本地 Python 环境、Node 环境、文件统计数量及网络连接，来快速定位瓶颈。

---

## 验证工作总结
- **我们不验证**：代码性能、单元测试、部署环境（因为 SuperClaude 是纯文本配置）。
- **我们验证**：文件是否存在、文件是否被正确读取、AI 的交互行为是否符合预期。

*只要文件齐备且可读，SuperClaude 通常就能正常工作。*
