# SuperClaude 故障排除指南 (Troubleshooting) 🔧

本指南提供从快速修复到深度诊断的各项操作，助您解决 SuperClaude 框架的相关问题。

## 快速修复 (解决 90% 的问题)

### 1. 安装核验
```bash
python3 -m SuperClaude --version    # 应显示 4.1.5
SuperClaude install --list-components
```

### 2. 命令无响应
如果在 Claude Code 中输入 `/sc:*` 命令没有反应，请尝试**完全重启** Claude Code 会话。

### 3. 检查清单
- [ ] 版本号是否正确（4.1.5）。
- [ ] 在 Claude Code 中输入命令是否有反馈。
- [ ] MCP 服务器是否正确列出。

## 常见问题处理

### 安装类问题
- **包安装失败**：尝试卸载后重新安装，并确保 `pip` 是最新版本。
- **权限不足 (PEP 668)**：推荐使用 `pipx` 安装；或使用 `pip install --user`；必要时使用 `chown` 修复 `~/.claude` 目录权限。
- **组件缺失**：使用 `--force` 参数强制重新安装所有核心、命令、智能体及模式组件。

### 命令与智能体问题
- **命令无法识别**：重启 Claude Code 并核验安装状态。
- **智能体无法激活**：尝试使用更具体的触发关键词，或通过 `@agent-name` 手动激活。
- **性能缓慢**：尝试使用 `--no-mcp` 禁用 MCP 服务器，或通过 `--scope` 限制扫描的文件范围。

### MCP 服务器问题
- **连接失败**：确保 `~/.claude.json` 配置文件存在，且已安装 Node.js 16+ 版本。使用 `--force` 重新安装 MCP 组件。
- **缺少 API Key**：如果使用了 Magic 或 Morphllm，请确保已正确导出相应的环境变量，或在命令后加上 `--no-mcp`。

## 深度诊断与重置
- 运行 `SuperClaude install --diagnose` 进行系统分析。
- 备份当前配置：`SuperClaude backup --create`。
- 彻底重置：先运行 `uninstall`，再执行全新的 `install`。
