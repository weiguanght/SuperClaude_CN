# SuperClaude 高级工作流合集 (Advanced Workflows)

本指南介绍了一系列复杂的命令序列与上下文组合，旨在帮助资深用户在 Claude Code 中高效管理大规模、多阶段的开发项目。

**核心认知**：这些是**对话模式**，而非自动执行的脚本。所有的“流程”都是通过在一个对话会话中循序渐进地叠加上下文来实现的，并不涉及真正的并行执行或自动化运维。

## 多维度项目实践

### 全栈开发序列
按照 `制定架构 -> 明确安全需求 -> 实施后端 API -> 设计 UI -> 前端对接 -> 整体质量分析` 的顺序，在单个会话中不断积累上下文，从而让后续的生成结果更具全局一致性。

### 复杂问题解决流
针对棘手 Bug，可以采用 `故障初步分析 -> 引入性能/架构专家深挖 -> 制定修复方案 -> 实施修复 -> 验证效果` 的结构化处理方式。

## 项目生命周期管理

### 项目初始化模式
- **探索阶段**：利用 `/sc:brainstorm` 挖掘非功能性需求。
- **规划阶段**：使用 `/sc:design` 产出系统架构，并由架构师智能体进行评审。
- **文档沉淀**：通过 `/sc:document` 开启文档化进程，并使用 `/sc:save` 总结当前关键决策。

### 渐进式迭代开发
通过对功能的模块化拆解（如：鉴权 -> 用户资料 -> 管理端），每一项新功能的实施都能直接受益于会话中已存在的旧功能上下文。

### 遗留系统迁移
1. 加载并深入分析旧系统。
2. 制定迁移假说与策略。
3. 建立兼容层并分步实施新组件。
4. 验证兼容性与业务无损。

## 质量保障与评审工作流

### 全方位代码评审
先后开启质量、安全、架构及性能分析，最后由 `/sc:improve` 一并应用修复建议。这种串行的专家会诊模式能显著提升代码的工业级水准。

### 测试驱动策略
在实施任何代码前，先通过 `/sc:design` 确定测试策略，随后由 `/sc:test` 生成不同层级（单元、集成、E2E）的测试代码。

## 会话管理最佳实践

- **保持会话专注**：不要在同一个对话中混杂不相关的项目任务。
- **控制会话长度**：当对话过长导致相应变慢或注意力分散时，先使用 `/sc:save` 总结现状，开启新会话并手动恢复关键上下文。
- **逻辑化排序**：遵循“分析 -> 设计 -> 实施 -> 测试”的自然开发逻辑。

---

*高级工作流的真谛在于精准的上下文层叠。通过在单次会话中合理排布命令与专家智能体，您可以引导 Claude Code 完成极其复杂的软件工程任务。*
