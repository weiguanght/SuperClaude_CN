# SuperClaude 高级模式 (Advanced Patterns)

本指南面向资深用户，介绍如何组合使用命令、智能体及标志位，以应对复杂项目中的上下文管理需求。

**请记住**：SuperClaude 为 Claude Code 提供上下文。所有的模式都是为了通过上下文引导 AI 的行为，而非直接执行代码或协调外部进程。

## 核心模式建议

### 1. 多智能体上下文组合
您可以先后激活多位专家，将他们的知识领域进行分层。
- **示例**：先通过 `@agent-security` 定义需求，再由 `@agent-backend-architect` 设计 API，最后通过 `/sc:implement` 进行实现。
- **原理**：上下文在 Claude 的理解中交织，从而生成更安全、更规范的代码。

### 2. 命令序列化工作流
- **渐进式精炼**：从全局 `/sc:analyze` 开始，逐步收窄至具体模块的 `--focus` 深度分析。
- **从探索到实施**：遵循 `脑暴 -> 设计 -> 专家评审 -> 实施 -> 验证` 的标准项目流。
- **持续改进**：结合 `/sc:analyze` 的分析结果与 `/sc:improve` 的修复，并由重构专家给出建议。

### 3. 混合标志位策略
- **深度控制**：根据任务难度选择 `--think` (标准型), `--think-hard` (深度型) 或 `--ultrathink` (竭尽型)；简易任务则可使用 `--uc` 压缩输出。
- **精选 MCP**：仅开启当前任务必需的 MCP 服务器（如 `--magic` 或 `--c7`），或在简单任务中使用 `--no-mcp` 以减少干扰。

## 典型场景应用

### 大型代码库分析
1. 理解结构（`--overview`）。
2. 识别高风险模块（由质量工程师智能体辅助）。
3. 针对性深挖（`--think-hard --focus quality`）。
4. 制定改进计划。

### 遗留系统迁移
1. 架构评估。
2. 识别现代化机会。
3. 制定迁移方案。
4. 带有兼容层的模块化实施。

## 高级使用原则

### ✅ 支持的操作
- **引导 AI 思维**：提供结构化的处理方式。
- **组合上下文**：叠加多个专业知识领域。
- **提升输出质量**：通过优质上下文带来更佳的生成结果。

### ❌ 不支持的操作
- **并行执行**：所有操作均为串行的上下文加载。
- **进程协调**：SuperClaude 不具备进程管理能力。
- **运行性能优化**：它不直接运行代码，因此不产生实际的运行性能影响。

### 最佳实践
- **有意识的分层**：按逻辑顺序添加上下文。
- **避免过载**：太多的智能体可能导致焦点模糊。
- **范围适中**：将复杂任务拆解为可管理的区块。

---

*高级 SuperClaude 模式的精髓在于精准的上下文管理和逻辑化的命令排序，帮助 AI 在理解深度与广度之间达成完美平衡。*
