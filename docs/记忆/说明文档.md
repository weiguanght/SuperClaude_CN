# 记忆数据目录 (Memory)

该目录存放 SuperClaude 框架 PM 智能体的记忆与学习数据。

## 概览

PM 智能体系通过多重记忆系统来学习、改进并在不同会话间维持上下文：
1. **反思记忆 (ReflexionMemory)** —— 错误学习与模式识别。
2. **工作流指标 (Workflow Metrics)** —— 性能追踪与优化。
3. **模式学习 (Pattern Learning)** —— 记录成功的开发模式。

## 文件说明

### reflexion.jsonl (自动生成)
- **用途**：错误学习数据库。
- **内容**：存储过去的错误、根因及解决方案，用于即时修复重现的失误。

### workflow_metrics.jsonl (自动生成)
- **用途**：任务性能追踪。
- **内容**：追踪 Token 消耗、执行时长及成功率，用于持续优化。详见 [工作流指标架构](WORKFLOW_METRICS_SCHEMA.md)。

### patterns_learned.jsonl (自动生成)
- **用途**：记录成功的开发模式，便于日后复用。

### 文档文件
- `WORKFLOW_METRICS_SCHEMA.md`: 指标数据的详细定义。
- `pm_context.md`: PM 智能体的上下文管理与 Token 效率策略。
- `token_efficiency_validation.md`: 效率优化的验证结果。
- `last_session.md`: 记录以往会话的重点内容。
- `next_actions.md`: 记忆系统的后续改进计划。

---

## 维护与安全

- **自动管理**：上述 `.jsonl` 文件由系统自动创建与维护，请勿手动编辑。
- **隐私保护**：本系统仅记录元数据（Token、耗时、任务类型）及通用错误逻辑，**严禁存储**密码、API 密钥或其他敏感个人信息。如果错误信息中包含敏感内容，请手动编辑文件进行脱敏处理。
- **Git 配置**：建议将 `reflexion.jsonl` 添加到 `.gitignore` 中（如果是个人开发者），或者将其包含在代码库中（如果是团队协作学习）。
- **性能**：在达到 10,000 条记录前，由于文件体积较小，检索性能通常维持在毫秒级。
