# SuperClaude 框架代码贡献指南 (Contributing) 🛠️

欢迎参与 SuperClaude 框架的开发！本指南将协助您理解如何通过编写结构化的提示词（Prompt）与 MCP 服务器集成，来增强 Claude Code 的行为能力。

## 核心架构概览
SuperClaude 是一个**面向上下文的配置框架**。它不是运行中的软件，而是 Claude Code 在对话时读取的指令文件集合。

```
SuperClaude_Framework/
├── superclaude/           # 框架核心组件 (真理之源)
│   ├── Core/             # 原则、规则与标志位定义
│   ├── Agents/           # 各领域的专家上下文 (如 security-engineer.md)
│   ├── Commands/         # 触发行为的命令上下文 (如 implement.md)
│   ├── Modes/            # 行为修改模式 (如 brainstorming)
│   └── MCP/              # MCP 服务器配置模板
```

## 贡献指南

### 1. 开发环境建议
- 确保已安装 **Claude Code**。
- 准备好 **Python 3.8+**（用于安装工具）及 **Node.js 16+**（用于 MCP 服务器）。

### 2. 编写上下文文件 (.md)
- **指令清晰**：为 Claude Code 提供简洁、可操作的行为指引。
- **使用元数据**：在文件开头使用 YAML Frontmatter（如 `name`, `description`）。
- **设置触发器 (Triggers)**：明确该专家或命令应在何时被激活。

### 3. 开发流程 (Git Workflow)
1. **Fork** 仓库并创建特性分支（`feature/your-feature`）。
2. **编写/修改** 组件后，将其手动复制到 `~/.claude/` 对应的目录下进行实测。
3. **提交 PR**：描述您的变更类型（Bug 修复、新功能、文档改进）以及测试情况。

### 4. 质量标准
- **可验证性**：指令必须能在实际对话中产生预期的行为变化。
- **无冲突**：确保新智能体或命令不会与现有组件产生逻辑干扰。
- **专业性**：使用规范的 Markdown 格式与专业的措辞。

## 许可协议
参与贡献即表示您同意将代码以 **MIT 协议** 授权，赋予社区最大的使用与分发自由。
