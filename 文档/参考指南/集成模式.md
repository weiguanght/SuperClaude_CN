# SuperClaude 集成模式库 (Integration Patterns)

本指南旨在向用户展示如何将 SuperClaude 命令、智能体与各类开发框架及工具进行高效集成。

**核心原则**：SuperClaude 告诉 Claude Code **该做什么**以及**该如何思考**。所有的代码生成均由 Claude 完成，SuperClaude 本身不直接运行代码。

## 典型框架实践

### 1. React 开发
- **模式**：`/sc:implement "React 应用需求" --c7`。
- **作用**：利用 `Context7` MCP 获取 React 最新文档（如果已配置），并结合 `magic` 标志位辅助 UI 组件开发。

### 2. Node.js 后端开发
- **模式**：`/sc:implement "Express API 需求" --c7`。
- **作用**：引导 Claude 遵循最佳的中间件实践与项目结构，进行 API 模式设计。

### 3. Python 开发
- **模式**：`/sc:implement "FastAPI 应用需求"` 后接 `@agent-python-expert`。
- **作用**：借助 Python 专家智能体审核代码风格及性能建议，并自动适配 pytest 模式。

### 4. 全栈联动
遵循 `脑暴架构 -> 智能体评审 -> 分别实施前后端 -> 实现前后端集成` 的标准流程。

## 常用工具协同模式

### 灵活运用 MCP 服务器
- **Context7**：获取各类框架的官方文档。
- **Sequential**：辅助处理复杂的逻辑推导任务（使用 `--seq` 标志）。
- **Magic**：推荐用于 UI 组件的现代化设计。
- **禁用 MCP**：在执行简单的工具函数编写时，使用 `--no-mcp` 以减少无关信息的干扰。

### 智能体与命令组合
- **安全驱动**：在 `/sc:implement` 之前，先让 `@agent-security` 审核鉴权需求。
- **质量保障**：在功能完成后，委派 `@agent-quality-engineer` 进行代码评审及测试用例生成。

## 重要说明

### ✅ 它们能做什么
- 为开发任务提供结构化的思考模板。
- 高效地叠加不同领域的专业知识上下文。
- 引导 Claude 生成更符合工业水平的代码。

### ❌ 它们不能做什么
- **不执行代码**：所有的“集成”都发生于 Claude 的文本理解中，不涉及真实的部署或运行。
- **不测量性能**：性能改进建议是基于知识而非实际运行数据。
- **无并行处理**：所有的上下文加载都是串行且相互独立的。
