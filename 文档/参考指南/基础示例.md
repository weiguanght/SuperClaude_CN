# SuperClaude 基础示例库 (Basic Examples)

本指南旨在为初学者提供开箱即用的“复制-粘贴”式命令参考，重点关注 SuperClaude 的核心用法与日常开发工作流。

**重要说明**：这些 `/sc:` 命令与 `@agent-` 调用会引导 Claude Code 加载特定的行为指令集。

## 核心命令精选

### 1. 需求调研：/sc:brainstorm
- **用途**：交互式项目探索与需求采集。
- **示例**：`/sc:brainstorm "一款用于健康追踪的移动应用"`。
- **反馈**：AI 将发起苏格拉底式的对话，挖掘需求、分析可行性。

### 2. 代码分析：/sc:analyze
- **用途**：针对现有代码库进行缺陷分析与改进评估。
- **示例**：`/sc:analyze src/ --focus security`。
- **反馈**：执行全面的安全审计，产出漏洞报告与修复建议。

### 3. 功能开发：/sc:implement
- **用途**：遵循最佳实践实现完整的功能。
- **示例**：`/sc:implement "集成 JWT 与频率限制的用户鉴权系统"`。
- **反馈**：编写完整的逻辑代码、安全校验与配套测试。

### 4. 故障诊断：/sc:troubleshoot
- **用途**：系统化地诊断并解决问题。
- **示例**：`/sc:troubleshoot "API 在用户登录时返回 500 错误"`。
- **反馈**：分步诊断、定位根因并给出多级解决方案。

### 5. 质量验证：/sc:test
- **用途**：为现有代码生成详尽的测试套件。
- **示例**：`/sc:test --focus quality`。
- **反馈**：产出全覆盖的测试用例与质量评估报告。

## 专家智能体手动激活
您可以不依赖自动触发，通过 `@agent-` 前缀精准启用特定领域的专家。
- `@agent-python-expert`：Python 性能优化与异步模式建议。
- `@agent-security`：OWASP 合规检查与漏洞修复建议。
- `@agent-frontend-architect`：响应式架构设计与无障碍性建议。
- `@agent-quality-engineer`：制定测试策略并生成 E2E 测试用例。

---

## 常用开发工作流模式

### 探索 -> 实施模式
先通过 `/sc:brainstorm` 梳理需求，再利用 `/sc:analyze --c7` 确定技术方案，最后由 `/sc:implement` 完成代码编写。

### 问题 -> 修复模式
先由 `/sc:troubleshoot` 定位根因，再通过针对性的 `/sc:analyze` 确定受影响范围，最后运行 `/sc:implement` 应用性能优化或 Bug 修复。

---

## 标志位 (Flags) 实用技巧

- **深度控制**：`--think` (标准型), `--think-hard` (进阶型)。
- **分析受限**：`--scope file` (仅分析单个文件)。
- **工具叠加**：`--c7` (引入官方文档参考)。
- **资源高效**：`--uc` (大型项目中开启 Token 压缩模式)。
